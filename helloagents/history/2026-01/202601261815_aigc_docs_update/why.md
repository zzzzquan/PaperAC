# 变更提案: AIGC判别规则与文档统一

## 需求背景
当前文档仍以LLM API为核心判别方式，且AIGC疑似分级规则未明确。为满足产品定义，需要将判别结果划分为高度疑似与轻度疑似，并按7:3比例在用户设定的AIGC占比x内进行分配。同时，判别算法方向调整为数学算法方案（暂不设计具体算法），因此需统一移除LLM接入相关表述并改为占位说明。

## 产品分析

### 目标用户与场景
- **用户群体:** 课程论文写作场景中的学生、助教、教学管理人员
- **使用场景:** 上传论文PDF，获得AIGC疑似程度分布与报告
- **核心痛点:** 需要可解释、可控的AIGC占比与分级结果

### 价值主张与成功指标
- **价值主张:** 提供可调占比与分级结果的AIGC检测，便于教学与审核
- **成功指标:** 文档一致反映AIGC分级与占比规则，LLM依赖描述完全移除

### 人文关怀
尊重学习者权益，明确检测用途为教学与审核，避免将结果用于惩罚性评判。

## 变更内容
1. 明确AIGC疑似分级规则：高度疑似/轻度疑似比例为7:3，合计占总字数x
2. 文档统一移除LLM API接入描述，改为数学算法占位
3. README 重写并明确未实现内容的占位说明

## 影响范围
- **模块:** AIGC判别、报告生成、API服务、架构设计、数据模型、项目规划
- **文件:** PLAN.md, helloagents/wiki/overview.md, helloagents/wiki/arch.md, helloagents/wiki/api.md, helloagents/wiki/data.md, helloagents/wiki/modules/aigc.md, helloagents/wiki/modules/report.md, README.md
- **API:** 文档层面调整（占位）

## 核心场景

### 需求: AIGC疑似分级与占比规则
**模块:** AIGC判别
按用户输入AIGC占比x（0-100%）生成疑似分级，分配为高度疑似7份、轻度疑似3份，合计占总字数x。

#### 场景: 占比控制
用户输入x后，报告中高度疑似与轻度疑似合计占总字数x，且比例为7:3。
- 输出高/低疑似占比与说明
- 总占比不超过x

### 需求: 去除LLM接入
**模块:** AIGC判别
判别算法不使用LLM API，改为数学算法方案占位。

#### 场景: 算法占位
文档中描述为数学算法，占位等待后续设计。
- 无LLM依赖
- 暂不定义具体算法
