# API服务

## 目的
统一入口、鉴权与任务编排。

## 模块概述
- **职责:** 登录鉴权、任务创建、流程编排、配额校验
- **状态:** 🚧开发中
- **最后更新:** 2026-01-28

## 规范

### 需求: 邮箱验证码登录
**模块:** API服务
提供邮箱验证码登录、会话与频控能力。

#### 场景: 发送验证码
请求发送验证码后记录发送与频控。
- 同邮箱60秒限频
- 同IP每小时20次

#### 场景: 校验登录
验证码有效期5分钟，最多5次错误，成功后一次性失效。
- 不存在用户自动创建
- 创建Redis会话并写入Cookie

### 需求: 任务编排
**模块:** API服务
负责从上传到报告生成的全流程编排。

#### 场景: 创建任务
创建任务后启动PDF处理、AIGC判别与报告生成。
- 记录任务状态与阶段
- 出错可重试或回滚

### 需求: 配额校验
**模块:** API服务
在任务创建前校验配额。

#### 场景: 配额不足
当配额不足时拒绝任务并提示升级。
- 不应消耗上传内容
- 记录失败原因

### 需求: 任务编排（MVP）
**模块:** API服务
提供上传、查询、列表与下载接口。

#### 场景: 上传创建任务
用户上传PDF后创建任务并入队。
- 校验PDF与大小限制
- 生成任务ID并返回

#### 场景: 查询任务
返回任务状态、进度与错误信息。
- 仅允许访问本人任务

#### 场景: 下载结果
仅success任务可下载结果文件。

## 依赖
- 账号与计费
- PDF处理
- AIGC判别
- 报告生成
