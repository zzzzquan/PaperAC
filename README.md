## 项目介绍

这是一个纯前端的玩具项目。

**本项目并非基于真实的深度学习模型。** 相反，本项目仅仅构建了一个关于句子分布的数学统计模型，来生成一篇 **AIGC率极低** 但 **“就像真的AIGC检测报告”** 的伪报告。

我始终认为，纯文本的 AIGC 检测在本质上是个伪命题：

从技术角度看，精准检测极难实现，误判情况层出不穷（实际上：1、即便存在一个完美的检测模型，我也可以基于此训练出一个能避开所有检测的生成模型。2、人类的写作是概率空间内的一种分布，而 AI 创作则是对这种分布的模仿与复刻。当 AI 的采样精度足够高时，两者的概率分布函数会趋于重叠。这样，你根本就无法从一个随机样本中反推它究竟来自哪个分布）

此外在实际应用中，很多检测场景也已异化为“检测学生是否认真”，而非“内容是否扎实严谨”，更不必说市面上无数打着检测旗号牟利的商业网站了。

我也能理解人们的担忧：AI究竟是在为我们赋能，还是在诱导我们放弃思考？AI生成的内容是否真正属于我们？这些问题在“AI写作”“AI审稿”大行其道的当下，没人能够说的清楚……但无论如何，AIGC检测这种可解释性差、甚至理论上根本不成立的技术，成为某些人偷懒甚至渔利的工具，属实有点说不过去吧……

**核心特性**：
- **开源免费**：项目采用完全开源策略，无需任何授权费用，支持自由使用与二次开发。
- **本地环境计算**：基于纯前端架构，所有文本解析、模型计算与报告渲染均在用户设备端（浏览器环境）本地独立完成。

## 本地部署

### 1. 环境准备
确保运行环境中已安装 [Node.js](https://nodejs.org/)（建议使用 v18.0.0 或更高版本）。

### 2. 构建与运行

1. 在项目根目录下执行依赖安装：
   ```bash
   npm install
   ```
2. 启动本地开发服务器进行调试：
   ```bash
   npm run dev
   ```
3. 访问本地服务：服务启动后，通过现代浏览器访问 `http://localhost:5173` 进行交互测试。

### 项目构建
```bash
npm run build
```
执行构建脚本后，项目将输出适用于生产环境的静态文件至 `dist/` 目录。该目录内容可直接对接标准 Web 服务器（如 Nginx、Apache）或静态云托管平台。
